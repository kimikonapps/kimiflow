## 🟢 Phase 7 — Code-review ensemble against specs → fix → commit-gate

0. **Frontend serialization preflight.** Before every Phase-7 entry/re-entry, run `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/frontend-quality-gate.sh .kimiflow/<slug> --write` (Codex: `$KIMIFLOW_PLUGIN_ROOT/hooks/frontend-quality-gate.sh`). Contract 1 requires OPEN plus exact `Frontend quality recovery: clean` / `Frontend quality recovery owns global: no` and no closed Recovery receipt. Legacy/read-only accepts only `OPEN reason=not-required` with all frontend keys/marker/sidecars absent. Otherwise return to Phase 5/6 without starting or advancing code recovery. If a code fix makes routing/evidence stale, re-enter Phase 5/6 while global code Recovery remains active; the frontend gate accompanies it as non-owner and never clears it.

0.5. **Conformance serialization preflight.** When STATE declares `Conformance contract: 1`, run `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/conformance-gate.sh .kimiflow/<slug>` (Codex: `$KIMIFLOW_PLUGIN_ROOT/hooks/conformance-gate.sh`) before every review entry/re-entry and require OPEN. Include the compact receipt plus decision checks in the existing Phase-7 review packet; do not add a reviewer. Any repair makes the content basis stale and therefore returns through Phase 6 before review resumes. After the named-path commit, `active-run.sh finish --write` uses terminal mode to require HEAD/index/worktree identity and repeats it transactionally after learning/outcome writes; a stale or different delivery cannot retire the run. Legacy/audit/review runs without the contract remain OPEN/not-required.

1. **Fixed review basis + compact packet.** Before each review round, resolve `review_target_sha` with `git rev-parse HEAD`, then validate and pin one basis for every axis in that round: user-supplied base ref when present; otherwise schema-4+ runs with local Red/verification checkpoints use the active session's immutable, STATE-backed `started_head`; otherwise use the repository default branch for committed branch work or `HEAD` for working-tree-only review. A repair commit requires a fresh round and freshly resolved target; the base remains immutable. Set `review_base_sha` to its merge-base with `review_target_sha` (or the target SHA for working-tree-only review). Compute the complete named run delta from committed `git diff <base>...<target>`, tracked worktree `git diff <target>`, staged state, and `git ls-files --others --exclude-standard -- <named paths>`; include new-file contents explicitly. Only this combined set being empty may end without reviewers. Record refs/SHAs, every exact command, reviewed commits, and untracked names in `CODE-REVIEW.md`; every reviewer receives the same basis and new-file payload. Discover and record `spec_sources` in precedence order (explicit user source → run-local `ACCEPTANCE.md` + `INTENT.md`/`PROBLEM.md` → issue/PR refs in reviewed commits → matching `docs/`, `specs/`, `.scratch/` material) and `standards_sources` (`AGENTS.md`/`CLAUDE.md`, `CONTRIBUTING.md`, `CODING_STANDARDS.md`, `.kimiflow/STANDARDS.md`, relevant architecture docs). If no spec exists, mark the Spec part unavailable instead of inventing intent; the correctness/regression part still runs against observable contracts. Build one compact packet: source references + only relevant excerpts, changed-file list, `git diff --stat`, relevant hunks plus complete named new files, diagnosis/repro artifacts when present, decisive Phase-6 evidence, and exact verification commands.
2. **Candidate axes in parallel.** Read → reference.md "Review rubric" (orchestrator only). Spawn narrow fresh-context `top`/`cross_family_top` reviewers with adversarial framing when those seats exist. If no stronger child or cross-family transport is available, the current strongest available orchestrator executes the same axis packets itself as separate fresh passes, records the quality fallback, and continues; lack of leaf routing never closes the gate. Deliberately selected `balanced`/`cheap` support seats never issue semantic review verdicts. **Reviewers do not read reference.md** — inline each spawn's axis line + the CANDIDATE grammar + file-form constraints (column 0, no newline in the claim; nothing but CANDIDATE lines or the single sentinel `NONE`) into the prompt. `small` runs use at least **two** axes (`quick` alias: review light — ONE `spec-correctness` axis); add the third when the diff touches hooks, plugins, memory, launcher, APIs/contracts, multiple surfaces, or any high-risk path. On `small`, fold documented standards into R2 only when R3 is not scheduled; `large`/release-critical uses all three. One axis (default `spec-correctness`) is **cross-family by default** when available — same transport/fallback as Phase 4 (→ reference.md "Model routing (per-role)"). The standard axes are:
   - **R1 spec-correctness:** trace requirements independently; find missing/partial/wrong behavior, unrequested scope, logic/edge regressions, and weakened/insufficient tests.
   - **R2 failure-security:** input validation, secrets/privacy, path handling, rollback/failure atomicity, partial writes; on `small` without R3, also apply the documented-standards/smell dimension.
   - **R3 standards-integration:** documented path-applicable project standards plus installed hooks/metadata/docs/launcher wiring, host parity (Claude/Codex), command/API/schema contracts; when Architecture Deliberation is active, verify the implementation still fits its operating envelope and selected invariant. Demand architecture change only with the executable falsifier or a concrete named-invariant violation; apply the compact smell baseline only as a heuristic.
   Each axis writes `.kimiflow/<slug>/code-review-candidates/r<N>-<axis>.md` using `CANDIDATE <SEVERITY> <ref> :: <claim> :: verify=<smallest check>` or `NONE`; when STATE declares Convergence Contract 1, the check is typed as `verify=command:<method>|verifier:<method>`.
3. **Orchestrator verification, axis-preserving.** Verify each candidate with targeted code reads, commands, or reproduction; for BLOCKER/HIGH candidates include an **active refutation attempt** (execute its `verify=` check, read the full code path — "could this be wrong?"): refuted → rejected, not promoted. Write `CODE-REVIEW.md` under separate `Spec / Correctness`, `Failure / Security`, and `Standards / Integration` headings with source status, candidates, evidence, and accepted/rejected/unverified outcomes. Keep cross-axis duplicates visible and linked in the report; do not rerank across axes, but promote an exact underlying defect only once with all applicable axis labels so the mechanical count is not inflated. Legacy runs promote `.kimiflow/<slug>/findings/r<N>-code-verified.md` as `FINDING <SEVERITY> <ref> :: [<axis-labels>] <reason>` or `NONE`. With Convergence Contract 1, assign the stable class from the violated AC/invariant, execute the typed candidate check, write a one-line regular non-symlink `review-evidence/` receipt, and promote only `outcome=reproduced` using the Phase-4 contracted FINDING grammar plus its SHA-256. A prior class that disappears requires the same method with `outcome=not_reproduced` plus the contracted `RESOLVED` row; `NONE` alone cannot clear it. The resolver counts only the promoted file, never raw candidates.
4. **Advisories.** First stage the run's named changed paths (`git add <named paths>` — never `-A`; the scans read the staged diff). Then run the bundled test-weakening scan `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/test-weakening-scan.sh` over the staged diff → append its `FLAG` lines to `.kimiflow/<slug>/ADVISORIES.md`; if it can't be located or run, note it in STATE — never silently skip the advisory channel. Likewise run `hooks/secret-content-scan.sh` and `hooks/lsp-diagnostics.sh` when present. A smell with neither a documented-standard violation nor demonstrable impact is advisory-only; skip issues enforced by existing tooling. The Simplicity dimension still writes `FLAG`s to advisories (folded into an existing lens for `small`; dedicated prosecutor for `large` or size-tripwire).
5. **Fix + autonomous recovery.** Fix confirmed BLOCKER/HIGH, then re-run affected axes and repeat verification; carry forward every still-applicable verified finding from an unaffected axis. Shared/uncertain changes rerun all scheduled axes. Use Phase 4's exact strategy-epoch contract with gate=`code`: `PLAN.md` is also the canonical code-strategy basis, so update it when the implementation strategy changes; before code round 1 append exactly one `kimiflow:strategy gate=code` baseline with its SHA-256. Let `S`, budget `B` (`quick`/`small`=2; `large`/release-critical=3), absolute `C=S+B-1`, globally increasing code-gate rounds, STATE `Review gate: code`, and resolve via `resolve-review-gate.sh ... --round <N> --expect code-verified --gate code --epoch-start <S> --cap <C>` plus `--finding-contract 1` when Convergence is declared. `incomplete|malformed|unproven-resolution` repairs candidate transport/evidence; `open-findings` gets the next targeted repair; `root-class-repeated|oscillation|reappeared|cap-reached` writes a code entry plus matching receipt whose chained `before` and recomputed `after=sha256(PLAN.md)` prove changed strategy bytes. Then return to diagnosis/implementation/verification, preserve fix Red/Green evidence, use bounded memory/current primary sources plus the smallest refuting check, change strategy, and start the next epoch without `await-user`. After two failed recovery epochs, use one independent recovery solver. On `OPEN/clean`, replace STATE `Recovery: active` with `Recovery: clean` and mark the entry clean before commit. Never waive a finding, reset/overwrite valid rounds, repeat build approval inside confirmed scope/risk, or add a routine technical pause. → reference.md "Review rubric".
6. **Atomic local commit.** Read → reference.md "Commit hygiene". Triage every advisory autonomously with evidence: dismiss non-impact with a recorded reason or promote it to a real finding and return to review. Show a short summary, `git status`, and `git diff --staged`; at `quiet`, keep it to decisive sentences. Snapshot the NUL-delimited staged paths before adding the exact named run-owned paths; classify every pre-existing foreign staged path and leave it staged and untouched. Require no unstaged bytes in the named paths, then commit only those pathspecs with `git commit --only -m "<message>" -- <named paths>`; verify the new commit contains exactly the intended paths and the foreign staged snapshot is unchanged. Schema 4+ does this automatically under the original build authority: no `git add -A`, no co-author/AI trailer, tests green; if verified Phase-5 checkpoints already contain the complete reviewed change, do not create an empty commit. Push, release, destructive migration, and external publication remain separately authorized. Schema 3 retains its legacy `await-user --kind commit` gate. **(large)** write the verified Phase-6 command to local/untracked `.kimiflow/test-gate` when absent. **(audit)** keep one verified slice = one reviewable diff = one local commit; never batch slices.
7. **Learning loop + project memory — required before `Status: done`.** Run `MR review-run --run .kimiflow/<slug> --write` after verification/review — it records the learning set in `.kimiflow/project/LEARNINGS.jsonl`, refreshes bounded memory/index/metrics/proposal/provider state, and returns a compact notification. `review-run` is **quality-gated**: too-short/generic learnings, missing evidence, or unsafe memory content block the write. If the run has no durable learning, use `MR review-run --run .kimiflow/<slug> --write --skip "<reason>"` so the skip is explicit. A verified architecture principle becomes a durable standard only when its exact applicability is known: call `MR standards record --scope <glob> --type <invariant|constraint|preference|heuristic|legacy> --rule <one-line> --evidence <ref> --write`; never infer a global scope. Otherwise keep it run-local or record the verified choice in Decisions. Then run `MR verify-run --run .kimiflow/<slug>`; a CLOSED result blocks completion, including stale/changed evidence fingerprints. Do not invoke outcome evaluation manually: `active-run.sh finish --write` evaluates the final strategy transactionally after these gates; `park|fail|abort` evaluate best-effort without delaying safe closure. User/workflow preferences go through `record --scope user` (local-only). Provider/Vault safety: neither `.kimiflow/` nor repo files store the key; prefetch/sync handoffs export only freshly verified current, non-private, non-security learnings; it never patches skills or writes external notes blindly — approved skill candidates become review-only drafts under `.kimiflow/project/SKILL-DRAFTS/`. Everything else (history search, usage/economics/usefulness metrics, `consolidate`/`propose` curation, the provider subcommands): → reference.md "Memory Router & Learning Loop".
8. **Project-map auto-refresh** (recommended, non-blocking). After Phase-6 verify, run `PMS refresh --changed` (`PMS` = `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/project-map-status.sh`; Codex: `KIMIFLOW_HOST=codex` + `KIMIFLOW_PLUGIN_ROOT`) so the sections of the files this run touched are re-hashed back to `current`, deleted members are pruned, new files under a known section prefix are adopted (+sha256), and each refreshed `.sh` section's symbol index is rebuilt. `NEW-FILE` lines are structure hints only — no auto-facts; durable learnings still flow through the gated learning loop above. Skip silently when there is no `.kimiflow/project/INDEX.json`. The Stop-hook map-staleness nudge (`hooks/map-staleness-nudge.sh`) is the safety net that surfaces any residual staleness. → reference.md "Project Map Bootstrap".
9. **Close state.** For phase-read runs, read/record Phase 7 before `hooks/active-run.sh finish --write`; finish checks fresh reads through Phase 7 and, when declared, the pinned valid Execution Contract journal, writes the automatic `OUTCOME-EVALUATION.json`, updates the local strategy ledger when evidence is promotable, then retires the session. Missing, malformed, symlinked, oversized, or selector-mismatched execution evidence fails closed and is repaired autonomously. After the local commit and learning review are open, set all phases done and `Status: done`. From the primary tree, `finish --write` also detects this run's registered noncurrent exceptional tree and invokes the guarded retirement internally; it succeeds only for the clean, unlocked, terminal, Kimiflow-owned tree, atomically archives the complete checkout plus its identity-matched Git metadata, includes both archive paths in `workspace_retirement`, and preserves its branch. `fail` and `abort` perform the same safe retirement; `park` retains the tree for resume. A dirty/changed target is reported `deferred` and remains untouched.
