## 🟤 Phase 6 — Verify against acceptance criteria (goal-backward)

Run each check, prove the goal, and persist full evidence in the run artifacts — full checklist (incl. the ✓/⚠/✗ marking scheme, cold-start trigger list, LSP tool selection): → reference.md "Verification". If resolved verbosity is `quiet`, chat shows only pass/fail + command/path for decisive checks; full output stays in files.

- **Run each criterion's method** and show the command + the decisive result line(s) — not full logs. At `quiet`, show only the command label/status and artifact path unless a failure needs the decisive error line.
- If the decisive method requires a clean tree, enter only through Phase 5's named local verification checkpoint and run it immediately; its failure loops to Phase 5 without user interaction.
- **Goal-backward:** for each criterion's artifact check Exists / Substantive / Wired — "task done ≠ goal achieved".
- **Fix mode (mandatory):** the reproduction no longer fails. Then run `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/red-green-gate.sh .kimiflow/<slug> --mode fix` (Codex: `KIMIFLOW_HOST=codex` and `KIMIFLOW_PLUGIN_ROOT`). `OPEN` is required before Phase 7, memory promotion, or `Status: done`.
- **Local diagnostics advisory:** when code changed, run `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/lsp-diagnostics.sh` and append any `FLAG` lines to `.kimiflow/<slug>/ADVISORIES.md` (bounded existing local tools only, never installs).
- **Regression:** existing/affected test suite green.
- **Architecture falsification (when active):** execute the PLAN's `Architecture check`, then compare the delivered diff and real integration/data-flow evidence with the recorded operating envelope and `fit|evolve|replace` decision. Under Conformance Contract 1, a failed falsifier or disproved invariant is `architecture_falsified` and returns to Phase 2 with changed evidence/strategy; legacy runs return to Phase 5. Unsupported reviewer taste does not steer either route.
- **Frontend contract (when declared).** Recompute the full Active-Run Git delta (committed, staged, unstaged, untracked, deletes and both rename sides), make STATE `Affected files` set-equal, recheck Lane/evidence, then run `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/frontend-quality-gate.sh .kimiflow/<slug> --record-routing --write` (Codex: `$KIMIFLOW_PLUGIN_ROOT/hooks/frontend-quality-gate.sh`). If routing reveals a new UI signal, return to Phase 2/5. For `off`, create no visual artifact and require the normal gate OPEN. For `standard|flagship`, read all of `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/references/frontend-quality-qa.md` (Codex: `$KIMIFLOW_PLUGIN_ROOT/references/frontend-quality-qa.md`), verify/correct the rendered implementation, write its exact `DESIGN-QA.md` plus current viewport-exact run-local PNG, then run the normal gate with `--write`. `Kind: contract` repairs only the reported contract; `Kind: visual` keeps screen identity fixed and changes strategy+pixels. CLOSED loops through Phase 5 without another user confirmation; OPEN alone advances.
- Cold-start smoke test when the diff touches boot-critical files; non-automatable criteria → a `top`/`cross_family_top` verifier subagent (derives pass/fail from evidence; does not trust the implementer's self-report). Mechanical commands and gates remain model-free.
- **(large) Independent verifier — additive:** one implementer-blind `cross_family_top` verifier when available, else a fresh same-family `top` verifier, re-derives the sweep and tries to falsify "done" claims. A discrepancy never bounces the run by itself — re-run the decisive command: confirmed → phase 5, else record the rejected claim and proceed.
- **Implementation conformance (Contract 1 only; adaptive, not a new phase).** Recompute the complete Git delta from STATE `Run started head` (including tracked, staged, untracked, deletes, and both rename sides), make STATE `Affected files` set-equal, and execute every PLAN `Check Dn` exactly. In `VERIFICATION.md`, write one exact `Decision check Dn: passed :: <PLAN method>` per row and compare the delivered paths/behavior with its evidence and invariant. Classify the result as `converged|code_gap|strategy_drift|architecture_falsified|research_stale|scope_drift`; refresh current external sources only for a named staleness/dependency/falsifier signal. `small` folds this judgment into the current orchestrator and makes no extra model call; `large` reuses the one independent verifier above, never adds another seat. Keep exactly one receipt: `<!-- kimiflow:conformance contract=1 status=<classification> diff=<passed|failed> strategy=<passed|failed> architecture=<passed|failed|not_applicable> research=<stable|stale|not_applicable> scope=<passed|failed> decisions=<N> checks=<N> verifier=<folded|independent> source=current-run -->`. For `converged`, require the generic outcome receipt below to be fully passed, then run `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/conformance-gate.sh .kimiflow/<slug> --record --write` (Codex: `$KIMIFLOW_PLUGIN_ROOT/hooks/conformance-gate.sh`) before setting Phase 6 done; after setting it done, the normal gate must be OPEN. Feed the exact non-converged classification to `active-run.sh next-action --event <classification>`: `code_gap|scope_drift` mechanically route to Phase 5, while `strategy_drift|architecture_falsified|research_stale` route to Phase 2 and change strategy/evidence without confirmation. The gate validates receipts/freshness and never executes PLAN commands.
- **Outcome receipt:** keep exactly one machine line in `VERIFICATION.md`: `<!-- kimiflow:verification outcome=<passed|failed> criteria=<passed|failed|not_run> regression=<passed|failed|not_run> -->`. Replace it after each verification attempt. `outcome=passed criteria=passed regression=passed` is allowed only after every required AC method and affected regression check is green; any other state stays failed/incomplete and cannot become a positive strategy outcome.
- Any verification failure not classified by Contract 1 → back to Phase 5 (escalation rule applies); the Contract-1 Phase-2 drift routes above take precedence.
