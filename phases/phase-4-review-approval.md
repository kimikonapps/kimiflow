## üü° Phase 4 ‚Äî Plan-gate (loop, binary, cap 3)

Read ‚Üí reference.md "Review rubric".

0. **Mechanical plan-blocker gate (before round 1):** run `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/plan-blocker-gate.sh .kimiflow/<slug>` (Codex: `KIMIFLOW_HOST=codex` and `KIMIFLOW_PLUGIN_ROOT`). `OPEN` is required before spawning reviewers. For phase-read runs, first record fresh reads through Phase 4; this gate checks through Phase 4. `CLOSED` means fix Phase-1/3 artifacts first; do not spend reviewer tokens on a run with a skipped small/quick micro-grill, unresolved markers, unmapped acceptance criteria, missing verification, missing path evidence, or undeclared affected files. (Audit runs are checked against the **audit profile** ‚Äî `AUDIT-INTENT.md` + `AUDIT.md` with slice path evidence, not `PLAN.md`/`ACCEPTANCE.md`.) Then perform the human-readable coverage check: every `ACCEPTANCE.md` criterion maps to a plan task AND a test; no orphan task lacks a criterion. Gaps ‚Üí fix the plan first.
1. Spawn **2 independent `top`/`cross_family_top` reviewers in parallel** (scope=small ‚Üí 1, lens B), fresh context, seeing only `PLAN.md` + `ACCEPTANCE.md` + `INTENT.md`/`PROBLEM.md` + named code (add `RESEARCH.md`/`DIAGNOSIS.md` only if the plan cites it; audit mode: `AUDIT-INTENT.md` + `AUDIT.md` + named code instead). `balanced`/`cheap` models never issue the plan-gate verdict. Frame each adversarially ("you did NOT write this; assume it is flawed; find the strongest objection" ‚Äî counters same-family self-preference). **Reviewers do not read reference.md:** inline into each spawn prompt (verbatim, <15 lines) the reviewer's lens definition (copied from the rubric), the FINDING grammar with its file-form constraints (one canonical line per finding, at column 0, no newline in the reason; nothing but FINDING lines, or the single sentinel line `NONE`), and the findings-file path. One lens is **cross-family by default** when a different-family CLI is available (`small` ‚Üí the single lens-B reviewer; `large` ‚Üí lens B) ‚Äî transport, verbatim findings persist, sticky fallback ‚Üí reference.md "Model routing (per-role)".
   - **A ‚Äî goal/completeness & understanding** (goal-backward; canonical definition ‚Üí reference.md "Review rubric").
   - **B ‚Äî risk:** security/edge/error/architecture/over-engineering (canonical definition incl. the fix-mode cause-not-symptom check ‚Üí rubric).
   - **(audit) refute the cut:** for each `delete`/`yagni` slice, actively hunt a **live caller** (repo-wide, incl. dynamic dispatch / reflection / string-keyed lookup). A cut survives only if no reviewer finds one; any live caller ‚Üí downgrade or move to do-NOT-touch. `shrink`/`stdlib` must preserve behavior (tests green before+after).
   - Each reviewer gives reasoning before verdict, then writes this round's findings to `.kimiflow/<slug>/findings/r<N>-<lens>.md` in the canonical one-line format (`FINDING <SEVERITY> <ref> :: <reason>`, sentinel `NONE` if clean ‚Äî ‚Üí reference.md "Review rubric"). No self-reported count; the orchestrator reads these files and never edits them.
2. Append a human-readable round summary to `REVIEW.md` (narrative only ‚Äî reasoning, not the gate truth).
3. **Gate (binary, NO numeric score):** count open BLOCKER/HIGH via `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/resolve-review-gate.sh .kimiflow/<slug>/findings --round <N> --expect <lensCSV>` (the tested source of truth; fail-closed; reason-codes ‚Äî ‚Üí reference.md "Review rubric"). 0 blocker/high ‚Üí gate open. Else revise narrowly, round +1 ‚Äî a finding is resolved only when the next round's reviewer no longer raises it (not asserted).
4. **Anti-oscillation (blocker-aware):** open BLOCKER/HIGH count doesn't strictly decrease across a round, or a disappeared finding reappears ‚Üí stop + ask, gate CLOSED (‚Üí reference.md "Review rubric").
5. **Cap (3) reached without an open gate ‚Üí stop + ask, gate CLOSED (never auto-proceed).**
6. **Gate open ‚Üí** `--prepare`: STOP, update STATE (0‚Äì4 done, `Status: backlog`), output `/kimiflow --resume <slug>`. Else ‚Üí step 7.
7. **Pre-build summary gate** (project-local, default on; control-flow only, never changes the engine ‚Äî toggle via `--settings`; ‚Üí reference.md "Pre-build summary gate"). `BG` = `${CLAUDE_PLUGIN_ROOT:-$CLAUDE_SKILL_DIR}/hooks/resolve-build-gate.sh`. `BG get`: `off` ‚Üí phase 5 ‚Äî EXCEPT the `full` alias: it fires regardless of the toggle (interactive; headless `full` parks as backlog). `on` ‚àß interactive ‚Üí print the **bounded pre-build summary** (structured, NOT a full-artifact dump): Problem/Goal ¬∑ Decisions ¬∑ Plan/Design ¬∑ Tests/Acceptance (incl. `AC-N ‚Üí test_name`) ¬∑ Risks ¬∑ Knobs (enabled knobs + planned Phase-5‚Äì7 fan-out, incl. the best-of-2 offer when its trigger holds) ¬∑ + artifact paths. At `quiet`, compress this stop to a few decisive sentences plus artifact paths; do not narrate planner/reviewer reasoning. Then **STOP**, ask "Approve to build, change something, or defer to backlog?" ‚Äî **approve ‚Üí phase 5**; **change ‚Üí phase 3** (revise ‚Üí re-gate); **defer ‚Üí backlog** (deliberately park the finished plan for later) ‚Üí STOP, update STATE (0‚Äì4 done, 5 open, `Status: backlog`), emit `/kimiflow --resume <slug>` ‚Äî the *explicit* twin of the `--prepare`/headless stop. `on` ‚àß headless / no answer ‚Üí do NOT build: behave like `--prepare` (STOP, update STATE `Status: backlog`, emit `--resume`).
